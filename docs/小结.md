# 简单Web服务器 - 实现总结

## 一、项目概述

使用 Go 语言标准库实现的多线程 Web 服务器，无第三方框架依赖。

- **技术栈**: Go 1.25 + 标准库 (net/http, bufio, os 等)
- **架构**: 模块化设计 + 双服务器架构
- **并发**: goroutine per connection

---

## 二、实现顺序与关键点

### 阶段1：TCP监听与双端口架构

```
管理面板服务器 (:8081) ← 始终运行，提供Web控制界面
Web服务器 (:8080)      ← 按需启动，提供文件服务
```

**关键代码**:
```go
// config/config.go - 配置单例
var cfg *Config
once.Do(func() {
    cfg = &Config{Port: 8080, AdminPort: 8081, RootDir: "/root"}
})

// router/router.go - 管理面板(标准库http实现)
mux := http.NewServeMux()
mux.HandleFunc("/static/", serveStatic)  // 静态文件
mux.HandleFunc("/api/", apiHandler)       // REST API
http.Server{Addr: ":8081", Handler: mux}.ListenAndServe()
```

### 阶段2：Web服务器启动与停止

**关键点**: context + select 优雅取消

```go
// server/server.go - 启动
listener, _ = net.Listen("tcp", addr)
ctx, cancel = context.WithCancel(context.Background())
isRunning.Store(true)
go run()  // 启动主循环

// 停止
isRunning.Store(false)
cancel()
listener.Close()
```

### 阶段3：请求处理主循环

**优雅停止：select + context 方案**

```go
// server/server.go - 使用 select 监听取消信号
func run() {
    for {
        conn, err := listener.Accept()
        if err != nil {
            select {
            case <-ctx.Done():
                return  // 优雅退出
            default:
                continue
            }
        }
        connections.Add(1)
        go func() {
            defer connections.Add(-1)
            handler.HandleConnection(conn)
        }()
    }
}
```

**停止流程**:
```go
isRunning.Store(false)  // ① 先设false
cancel()                  // ② 取消context
listener.Close()          // ③ 关闭listener
// ctx.Done() 立即关闭，goroutine收到信号直接return
```

### 阶段4：HTTP请求解析

```go
// handler/handler.go - 解析请求行和头部
reader := bufio.NewReader(conn)
line, _ := reader.ReadString('\n')
parts := strings.Split(strings.TrimSpace(line), " ")
method, url := parts[0], parts[1]

// 读取头部
headers := make(map[string]string)
for {
    line, _ := reader.ReadString('\n')
    if strings.TrimSpace(line) == "" { break }
    if idx := strings.Index(line, ":"); idx != -1 {
        headers[strings.TrimSpace(line[:idx])] = strings.TrimSpace(line[idx+1:])
    }
}
```

### 阶段5：文件读取与响应

**安全检查**: 路径遍历防护 + 目录边界检查
```go
// 防止 .. 路径遍历
if strings.Contains(url, "..") { return 403 }

// 确保文件在rootDir内
absRoot := filepath.Abs(rootDir)
absPath := filepath.Abs(filePath)
if !strings.HasPrefix(absPath, absRoot) { return 403 }
```

**响应构造**:
```go
response := fmt.Sprintf(
    "HTTP/1.1 200 OK\r\n"+
    "Content-Type: %s; charset=utf-8\r\n"+
    "Content-Length: %d\r\n"+
    "Content-Disposition: inline\r\n"+
    "Connection: close\r\n"+
    "\r\n", contentType, len(content))
conn.Write([]byte(response))
conn.Write(content)
```

### 阶段6：Content-Type 与 Content-Disposition

**显示 vs 下载**:
- 文本文件(.html/.css/.js/.svg/.json等): `inline` → 浏览器显示
- 二进制文件: `attachment` → 下载

### 阶段7：日志系统

**生产者-消费者模式**:
```go
// logger/logger.go
logChan := make(chan AccessLog, 100)  // 有缓冲channel

// 生产者(handler)
logChan <- AccessLog{ClientIP: ip, URL: url, Status: 200}

// 消费者(后台goroutine)
for log := range logChan {
    logs = append(logs, log)
    fmt.Printf("[%s] %s %s -> %d\n", ...)
}
```

## 三、模块结构

```
├── main.go           # 入口，初始化各模块
├── config/           # 配置管理(单例+读写锁) + Content-Type公共方法
├── server/           # Web服务器核心(启动/停止/主循环Accept委派)
├── handler/          # HTTP请求处理(解析/文件/响应)
├── logger/           # 访问日志(channel异步)
├── router/           # 管理面板路由(标准库http)
├── static/           # Web控制面板前端
└── www/              # 演示站点文件
```

## 四、核心架构理解（重要）

### 4.1 模块职责划分

| 模块 | 职责 | 理解要点 |
|------|------|----------|
| **logger** | 日志容器服务 | 暴露 Add/Get/Clear 接口，解耦日志业务 |
| **router** | 管理面板路由 | 处理 8081 端口的 HTTP 请求，全部返回 JSON |
| **server** | 文件服务器生命周期 | 管理 8080 端口服务器的启动/停止/配置，**不处理业务** |
| **handler** | HTTP 请求业务处理 | 解析请求 → 安全检查 → 读文件 → 构造响应 |

### 4.2 server 和 handler 的分工

**server.go 的定位**：

- `net.Listen("tcp", addr)` = **服务器启动**的本质（TCP监听建立）
- `run()` = 连接管理器：不断 Accept，有**客户端连接**就丢给 handler 处理

**handler.go 的定位**：

- 处理 HTTP 业务逻辑：解析请求行/头部、路径安全检查、读文件、构造响应

```
客户端浏览器
    │
    │ HTTP请求 (GET /index.html)
    ▼
server.go: listener.Accept()  ← 接收连接
    │
    │ 交给 handler 处理
    ▼
handler.go: HandleConnection(conn)
    │
    ├── 解析请求行（GET /index.html HTTP/1.1）
    ├── 解析请求头部
    ├── 安全检查（路径遍历防护）
    ├── 读取文件（/root/index.html）
    ├── 构造HTTP响应（状态行+头部+body）
    │
    ▼
返回响应给客户端
```

**关键理解**：`server.go` 只管"接活"，真正的"干活"是 `handler.go`

## 六、关键问题与解决

### 1. 停止服务器死循环
**原因**: 原方案依赖 `isRunning` 标志判断退出，存在竞态条件
**解决**: 使用 `select` 监听 `ctx.Done()`，context取消时立即退出，优雅且无竞态

### 2. 文本文件中文乱码
**原因**: CSS/JS的Content-Type缺少charset
**解决**: `text/css; charset=utf-8`

### 3. 输入框被自动刷新覆盖
**原因**: 2秒刷新重置input.value
**解决**: focus时跳过刷新
